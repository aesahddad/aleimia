# 🏰 وثيقة هندسة "قلعة العينية" (Aleinia Castle Architecture)

> "الحقيقة المرة أفضل من الترقيع الجميل."

تحدد هذه الوثيقة الحالة المعمارية الحالية لمنصة العينية، والديون التقنية (Technical Debt)، وخارطة الطريق للتحول إلى بنية احترافية عالمية.

## 🔴 الحالة الحالية (The Bitter Truth)
المشروع يعاني من عدة "كوارث" برمجية يتم العمل على تصفيتها:
1.  **ديكتاتورية النواة**: ملف `App.js` يحمل مسؤوليات أكثر من طاقته.
2.  **الارتباط العضوي القاتل**: الميزات مرتبطة ببعضها مباشرة، مما يجعل تغيير أي جزء يخاطر بانهيار الأجزاء الأخرى.
3.  **الأمان الهش**: نظام الصيانة والمصادقة كان يحتوي على ثغرات منطقية تسمح بتجاوز الصلاحيات.
4.  **فوضى البيانات**: غياب الفهارس (Indexes) واستخدام الـ Regex في البحث على قاعدة بيانات كبيرة يهدد الأداء.

## 🏛️ المبادئ المعمارية الجديدة (The Castle Way)
نحن ننتقل الآن إلى نموذج **Clean Architecture** المعتمد على:

### 1. طبقة الخدمات (Service Layer)
- فصل منطق الأعمال (Business Logic) عن المسارات (Routes).
- مثال: `SearchService.js` الذي يفصل منطق البحث عن الـ API.

### 2. الجهاز العصبي المركزي (EventManager)
- فك الارتباط بين الأجزاء باستخدام الإشارات (Events).
- لا يجب أن يعرف الـ API شيئاً عن الـ UI؛ هو فقط يرسل إشارة "صيانة"، ومن يهتم يقوم بالرسم.

### 3. تجزئة الواجهة (UI Modularization)
- تقسيم ملفات الـ UI الضخمة إلى مكونات (Components) صغيرة تحت مجلدات `ui/`.
- مثال: تقسيم `AdminUI.js` إلى `StoreUI.js` و `AdUI.js`.

### 4. الإدارة المركزية للأخطاء (Centralized Logging)
- استخدام `Winston` لتسجيل كل ما يحدث في القلعة في ملفات `logs/`.

## 🚀 الخطوات القادمة (Roadmap)
1.  **Authentication Hardening**: إضافة Refresh Tokens و CSRF Protection.
2.  **DB Optimization**: إضافة فهارس (Indexes) حقيقية لسرعة البحث.
3.  **UI Components Expansion**: استكمال تجزئة بقية الواجهات (الأعضاء، الإحصائيات، الحراج).
4.  **Validation Layer**: إضافة `Joi` أو `Zod` لفحص كل مدخلات الـ API.
